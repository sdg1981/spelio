# Spelio Welsh Spelling Basics — Educational Content Principles

## Purpose of this document

This document ONLY applies to the:

- Welsh Spelling Basics section
- introductory phonetic/sound explanation pages
- beginner educational walkthrough content
- support practice linked from those pages

It does NOT apply to:
- the core practice engine
- normal word-list progression
- admin tools
- analytics
- marketing pages
- future grammar/course systems

The goal of this subsystem is:

> to help adult learners feel that Welsh spelling becomes more understandable and predictable through listening, pattern recognition, and calm repetition.

This is NOT intended to be:
- a full Welsh course
- formal phonetics training
- linguistic instruction
- grammar-heavy teaching

The teaching philosophy prioritises:
- intuition
- pattern familiarity
- confidence
- recognition
- repeated exposure

over:
- technical completeness
- formal linguistic accuracy
- exhaustive rule teaching

---

# Core educational principles

## 1. Teach ONE main idea per page

Each page/card should teach:
- one recognisable sound pattern
- one conceptual insight
- one beginner mental model

Avoid:
- combining multiple unrelated ideas
- teaching several exceptions at once
- mixing concepts together

Example mistake:
- combining W and Y together under a misleading “wy” topic

Correct approach:
- separate W page
- separate Y page

---

## 2. Never introduce unexplained patterns inside an explanation

Do not use:
- unexplained Welsh sound combinations
- advanced patterns
- unintroduced concepts

inside beginner explanations.

Example mistake:
- using `ddwy`
- then breaking it into `dd • wy`
before `wy` had been properly taught

Correct approach:
- use examples that only contain already-taught or self-contained patterns

---

## 3. Every example must reinforce the exact lesson being taught

Example words must directly support the educational concept.

Incorrect:
- using a word without the target sound/pattern
- using examples that accidentally teach something else

Example mistake:
- using `brawd` on the dd page even though it does not contain `dd`

Correct:
- all example words must clearly demonstrate the target pattern

---

## 4. Use English sound anchors carefully

English comparisons are helpful, but they must:
- remain approximate
- avoid sounding absolute
- avoid overpromising precision

Avoid:

```text
Usually sounds like...
```

Prefer:

```text
Can sound a little like...
```

Examples:
- Welsh `w` → similar to “oo” in food
- Welsh `ch` → similar to Scottish “loch”
- Welsh `y` → may vary by word

Do not imply:
- exact sound equivalence
- fixed pronunciation rules

---

## 5. Always explicitly label English reference words

When referencing English sound examples:
ALWAYS specify:

```text
English word
```

Reason:
Direct translation into Welsh can otherwise accidentally imply:
- the Welsh example word itself explains the sound
- or that the learner should already understand Welsh pronunciation

Example mistake:

Incorrect:

```text
Sounds like hot
```

Correct:

```text
Sounds like the English word “hot”
```

This rule is ESPECIALLY important in Welsh translations.

---

## 6. Welsh translations must preserve educational meaning, not literal wording

Translations should preserve:
- learner understanding
- educational clarity
- contextual meaning

NOT:
- literal sentence structure

Common mistake:
directly translating English instructional phrases into Welsh in ways that lose context or become circular.

Example mistake:
Welsh text referencing Welsh example words as pronunciation anchors without clarifying they are English comparison words.

Correct:
Explicitly identify:
- English comparison words
- Scottish comparison words
- reference-language examples

inside Welsh copy.

---

## 7. Prefer recognition over technical explanation

Do not overload pages with:
- phonology
- syllable stress rules
- linguistic terminology
- advanced pronunciation systems

Correct beginner framing:

```text
Once you start hearing this pattern...
```

Avoid:

```text
This phoneme behaves differently in stressed syllables...
```

Spelio should feel:
- calm
- approachable
- confidence-building

---

## 8. Avoid duplicate lesson concepts across adjacent cards

If one card demonstrates:

```text
tan / tân
```

then the next card should NOT repeat the same core teaching point.

Each card should progress naturally.

Example correction:
- accents topic reduced from 3 cards → 2 cards
- because Card 1 and Card 3 duplicated the same idea

---

## 9. Keep support practice tightly aligned to the lesson

Support practice must directly reinforce:
- the exact sound pattern being taught

Incorrect:
- broad or mixed support lists
- practice words that contradict the page concept

Example mistake:
- “wy” support page mixing unrelated W/Y concepts

Correct:
- support_w
- support_y

as separate practice systems.

---

## 10. Audio-first learning matters more than rule memorisation

Pages should encourage:
- listening
- noticing
- recognition

rather than:
- memorising abstract rules

Good phrasing:

```text
Listen carefully...
```

```text
Once you start hearing...
```

Avoid:

```text
Memorise this rule...
```

---

# Content tone guidelines

The Welsh Spelling Basics subsystem should feel:

- calm
- intelligent
- lightweight
- adult-focused
- confidence-building
- pattern-oriented

Avoid:
- childish teaching tone
- over-enthusiasm
- gamified language
- academic linguistic tone
- long grammar explanations

---

# UI/content structure guidelines

Preferred page structure:

1. Symbol / visual anchor
2. Clear title
3. Short explanation
4. Listen-to-examples block
5. Optional tip (ONLY if genuinely useful)
6. Practice CTA (ONLY once learner is ready)

Tips should:
- add new understanding
- not repeat the main body text

If a tip repeats the explanation:
REMOVE IT.

---

# Practice CTA rules

Do not show:
“Practise this pattern”

until:
- the learner has enough understanding
- the examples/audio establish the concept first

Example correction:
- accents Card 1 removed practice CTA
- practice moved to later card only

---

# Beginner simplification rules

Prefer:
- helpful mental models
- recognisable patterns

over:
- strict linguistic completeness

Example:
Treating:

```text
dd + wy
```

as a learner-friendly chunk may sometimes be acceptable,
but only if:
- `wy` has already been taught
- the learner is not being introduced to unexplain
